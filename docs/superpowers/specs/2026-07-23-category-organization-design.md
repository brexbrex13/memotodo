# 設計：タスク一覧の整理（カテゴリ導入＋レイアウト刷新）方向性決定

> 本書は brainstorming の成果物（design spec）。次段階は writing-plans による実装プラン作成。
> 対象：`main` 起点。関連ドキュメント＝全体設計 `docs/superpowers/specs/2026-07-03-react-migration-design.md`。

## 1. 背景・課題

- タスクが増えてくると一覧がごちゃつき、見づらくなる。
- タスクにはいくつかのまとまり（カテゴリ）があり、カテゴリごとに整理したい。
- ただし本アプリの設計思想は「付箋のようにさっと登録できる TODO アプリ」。
  **登録動線にカテゴリ選択を割り込ませて登録の速さを損なうことは避ける**。
- あわせて、ヘッダー・タブ・OSタイトルバーが占める領域を削り、
  **できるだけ視覚的にシンプルかつ明瞭な UI** にしてタスク表示領域を広く取る。

## 2. 確定した設計判断（brainstorming 合意）

### カテゴリ

1. **方式＝Inbox＋あとから仕分け**。登録動線は現行のまま一切変えない。
   新規タスクは「未分類」に入り、分類は後から行う。
2. **カテゴリへの直接追加 UI も用意する**。仕分け専用にせず、カテゴリを見ながら
   そのカテゴリへ直接タスクを追加できる動線を持つ（§6.3）。
3. **見せ方＝上部フィルタチップ＋スティッキーなカテゴリ見出し**。
   - 一覧上部にカテゴリのフィルタチップを並べ、クリックでそのカテゴリのみ表示。
   - 「すべて」表示時はカテゴリごとのセクション表示とし、縦長になる前提で
     スクロール時にカテゴリ名見出しが上部にスティッキーする。
   - タブ（ボード）案・`#タグ` 記法案は不採用。
4. **1タスク1カテゴリ**（複数タグは不採用）。色付きラベルを1つだけ付けられる
   シンプルなモデルとし、付箋アプリの軽さを保つ。

### レイアウト刷新

5. **アプリヘッダー（MemoTodo バー）と「タスク一覧/完了済み」タブ行を廃止**し、
   画面構成を「ミニタイトルバー → 文字入力ボックス → フィルタチップ → タスク一覧」の
   4段に簡素化する。
6. **OS タイトルバーも廃止（フレームレス化）**。技術的懸念は §8 のとおり解消済み。
7. **ミニタイトルバーのボタンは「完了済みトグル・設定・×」の3つのみ**（最小化ボタンは置かない）。
   × は現行どおりトレイへの格納。
8. **「期日なし／期日あり」の2カード構成は廃止**し、並び順は
   **期日なし（先頭・手動並び替え維持）→ 期日あり（期日の近い順）** に統一する。
   「期日なし」はフィルタチップとしても絞り込めるようにする。

## 3. データモデル

```sql
CREATE TABLE IF NOT EXISTS categories (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    name       TEXT    NOT NULL,
    color      TEXT    NOT NULL DEFAULT '',   -- プリセットからの選択を想定
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at TEXT    NOT NULL
);
-- todos に category_id 列を追加（NULL = 未分類）
ALTER TABLE todos ADD COLUMN category_id INTEGER;
```

- `category_id` が NULL のタスクは「未分類」。既存データはマイグレーション不要で
  そのまま全件未分類として扱える。
- カテゴリ削除時は所属タスクの `category_id` を NULL に戻す（タスクは消さない）。
- 定期タスク（`recurring_tasks`）は今回スコープ外（必要になったら別途検討）。

## 4. 画面レイアウト（全体構成）

```
┌──────────────────────────────────────┐
│ ミニタイトルバー   [✓完了済み] [⚙] [×] │ ← ドラッグ移動領域を兼ねる
├──────────────────────────────────────┤
│ 文字入力ボックス（QuickInput）          │
├──────────────────────────────────────┤
│ フィルタチップ すべて|未分類|仕事|…|期日なし|⚙ │
├──────────────────────────────────────┤
│ タスク一覧（スクロール領域）             │
└──────────────────────────────────────┘
```

- **ミニタイトルバー**：`--wails-draggable: drag` でウィンドウ移動を担う細いバー。
  ダブルクリックで最大化/復元（`WindowToggleMaximise`）。
  - 「✓完了済み」：完了済み表示のトグル（現行タブの置き換え。表示中は入力ボックス・
    チップを隠し、現行どおり完了日時の新しい順で一覧表示）。
  - 「⚙ 設定」「×（トレイへ格納）」：現行機能の移設。× は既存の
    `OnBeforeClose`（トレイ常駐）をそのまま呼ぶ。
- **アプリヘッダー・タブ行は削除**。未完了件数バッジはチップ「すべて」に表示する。
- 画面右端の定期タスクタブ（RecurringTab/Panel）は現行のまま。

## 5. タスク一覧の仕様

### 5.1 フィルタチップ

- 構成：`すべて｜未分類｜<カテゴリ…>｜期日なし｜⚙`（単一選択）。
- カテゴリチップには色と件数バッジを表示。選択状態は UI ストアで保持。
- 「期日なし」は期日未設定タスクのみを横断表示する絞り込み（カテゴリ横断）。
- 末尾の「⚙」からカテゴリ管理（追加・リネーム・色変更・並び替え・削除）を開く。

### 5.2 並び順（全表示共通の原則）

- **期日なし（先頭）→ 期日あり（期日の近い順）**。
- 期日なし同士は現行踏襲の手動並び替え（ドラッグ）を維持。期日ありは期日順固定で
  ドラッグ不可。
- 「期日なし／期日あり」のカード分割・セクション見出しは廃止し、1つの流れで表示する
  （境界は罫線などの控えめな区切りに留める）。

### 5.3 「すべて」表示（カテゴリグルーピング）

- カテゴリ `sort_order` 順にセクションを並べ、最後に「未分類」を置く
  （Inbox として常に見える位置に残す。空のときは非表示）。
- **カテゴリ名見出しは `position: sticky` でスクロール中も上部に固定**。
- セクション内の並びは §5.2 の原則どおり。
- 各セクションは折りたたみ可能とし、折りたたみ状態は設定に永続化する。

### 5.4 絞り込み表示

- チップ選択時はそのカテゴリ（または未分類・期日なし）のタスクのみをフラットに表示。
  並びは §5.2 の原則どおり。

### 5.5 行・詳細・完了済み

- 行(TodoRow)にはカテゴリ色の小ラベル（ドット or チップ）を表示。
- カテゴリの付け替えは詳細（インライン／モーダル）にセレクトを追加して行う。
- 完了済み表示はグルーピングせず現行の時系列一覧のまま。行にカテゴリラベルのみ表示。

## 6. 登録動線（設計思想の維持）

### 6.1 QuickInput は不変

- 「すべて」「未分類」「期日なし」表示中の QuickInput からの登録は従来どおり
  未分類・期日なしで入る。入力欄・キー操作・見た目に変更なし。

### 6.2 カテゴリ絞り込み中はそのカテゴリへ登録

- カテゴリで絞り込み中は、QuickInput からの登録が**表示中のカテゴリに直接入る**。
  「いま見ている付箋ボードに貼る」という自然な拡張であり、登録操作自体は変わらない。
- 誤解を防ぐため、絞り込み中は QuickInput のラベル／placeholder に登録先カテゴリ名を表示する。

### 6.3 セクションからの直接追加

- 「すべて」表示時、各カテゴリセクションの見出しに「＋」を置き、クリックで
  セクション内にインライン入力を開いてそのカテゴリへ直接追加できる。

## 7. 仕分け操作

- 基本動線：詳細画面のカテゴリセレクトで付け替え。
- 追加動線（後続フェーズ）：行をカテゴリセクション（または折りたたみ中の見出し／チップ）へ
  ドラッグ＆ドロップして付け替え。既存の dnd-kit 基盤を流用する。

## 8. フレームレス化の技術検証結果

Wails v2（Windows）で以下を確認済み。**採用にあたっての技術的懸念なし**。

- `options.App{ Frameless: true }` で OS タイトルバーを削除できる。
- フレームレスでも**端ドラッグでのリサイズ・Aero スナップ・最大化は維持される**
  （Windows は WS_THICKFRAME 相当の挙動を Wails が処理）。
- ウィンドウ移動は CSS `--wails-draggable: drag` を指定した領域（ミニタイトルバー）で可能。
  行・ボタン等の操作要素には `--wails-draggable: no-drag` を指定する。
- タイトルバーのダブルクリック最大化は標準では失われるため、ミニタイトルバーの
  dblclick で `WindowToggleMaximise` を呼んで再現する。
- ×（トレイ格納）はフロントから `runtime.Quit()` を呼べば既存の `OnBeforeClose`
  （状態保存→`WindowHide`→トレイ常駐）がそのまま機能する。
- ウィンドウサイズ・位置の保存/復元（`main.go` の `saveWindowState`）への影響なし。

## 9. フェーズ分け

| フェーズ | 内容 |
|---|---|
| Phase 1（レイアウト刷新） | フレームレス化＋ミニタイトルバー（完了済みトグル・設定・×・ドラッグ移動・dblclick最大化）、ヘッダー/タブ行の廃止、期日なし/ありカード廃止と並び順統一（期日なし先頭・手動順 → 期日あり近い順） |
| Phase 2（カテゴリ基盤） | DB・API（categories CRUD／todos.category_id）、フィルタチップ（期日なしチップ含む）、カテゴリグルーピング＋スティッキー見出し、行のカテゴリラベル、詳細でのカテゴリ変更、カテゴリ管理 UI |
| Phase 3（直接追加・整理支援） | 絞り込み中の QuickInput 登録先切替、セクション「＋」インライン追加、セクション折りたたみ＋永続化 |
| Phase 4（任意） | D&D によるカテゴリ間移動、未分類のまとめて仕分け支援 |

## 10. スコープ外（今回は見送り）

- 複数タグ・タグ検索。
- 定期タスクへのカテゴリ付与。
- 「重要タスクの上部固定」「長期間未着手タスクの自動退避」
  （カテゴリ整理で不足を感じたら再検討）。
- ミニタイトルバーへの最小化ボタン設置（Win+↓／タスクバークリックで代替可能。
  不便であれば後から追加は容易）。
